## 第一层：技术可行性

### API 概况
- **官方 API**: LinkedIn Marketing API + Community Management API
- **文档地址**: https://developer.linkedin.com/
- **Posts API 端点**:
  - 创建帖子: `POST https://api.linkedin.com/v2/posts`
  - 上传图片: `POST https://api.linkedin.com/v2/images`
  - 上传视频: 需要分片上传
- **功能支持**: 文本帖、图片帖、视频帖、文档帖（轮播 PDF）、文章（Article）

### 认证方式
- **OAuth 2.0 三腿授权** (3-Legged OAuth Flow)
- 需要在 LinkedIn Developer Portal 创建应用
- 必须关联一个 LinkedIn Company Page 进行验证
- 所需 Scope:
  - `w_member_social` - 发布个人帖子
  - `w_organization_social` - 发布公司页面帖子
  - `r_liteprofile` - 读取基本资料

### Token 管理
- **Access Token 有效期**: 60 天
- **Refresh Token**: 支持，有效期 365 天
- 需要实现自动刷新机制
- Token 存储建议：本地加密存储，不用第三方服务

### 免费额度
- **无明确调用限制**，但有 rate limiting:
  - 大多数端点：100 次/天/用户
  - 发帖：无官方公开限制，建议 < 3 帖/天避免被视为垃圾
- **无需付费**：个人开发者可免费使用 API

### 能发什么 / 不能发什么
**能发**:
- 纯文本帖（最长 3000 字符）
- 图片帖（最多 20 张）
- 视频帖（最长 10 分钟）
- 文档帖 / 轮播（PDF 格式，最多 300 页）
- 文章（长文，无字数限制）
- 投票

**不能 / 限制**:
- 无法通过 API 发送 InMail
- 无法直接发布到别人的 feed
- 视频需要预处理 + 分片上传
- 无法编辑已发布帖子（API 不支持）

## 第二层：平台规则与最佳实践

### 算法偏好 (2025-2026)

LinkedIn 2025年底开始重大算法转变：**从"活跃度优先"转向"相关性优先"**

#### 核心变化
1. **相关性 > 时效性**: 算法会显示几天甚至几周前的高质量内容
2. **质量评估前置**: 帖子发布后先经过质量检测，再进入初始分发
3. **深度互动优先**: Saves 和 Sends 权重大幅提升
4. **个人账号 >> 公司页面**: 公司页面平均只能触达 1.6% 粉丝

#### 2026 年互动信号权重

|-----|-----|
| 保存 (Save) | ⭐⭐⭐⭐⭐ 新增指标，极高价值 |

### 内容格式甜区

#### 文字长度
- **Hook（前 2-3 行）**: 最关键，决定是否被展开阅读
- **正文**: 200-800 字最佳
- **最大**: 3000 字符
- **技巧**: 多用换行、emoji 增加可读性

#### 轮播 PDF
- **页数**: 8-15 页最佳
- **每页**: 1 个核心观点 + 视觉化
- **格式**: 1080×1350 或 1080×1080
- **首页**: 必须有吸引力的标题

#### 视频
- **时长**: 30秒 - 3分钟
- **比例**: 1:1 或 4:5（移动端友好）
- **字幕**: 必须，85% 用户静音观看

### 标签策略

**2025-2026 hashtag 最新规则：**
- **数量**: 3-5 个最佳（不要超过 5 个）
- **位置**: 放在帖子末尾，不要插入正文
- **组合**:
  - 2-3 个宽泛标签（如 #AI, #MachineLearning）
  - 1-2 个细分标签（如 #MLOps, #AIAgents）
- **避免**: #Business, #Success 等过于泛滥的标签

**有效标签示例** (AI 领域):
```
#ArtificialIntelligence #GenerativeAI #AIEngineering #TechFounder #StartupLife
```

### 降权行为

⚠️ **这些行为会导致帖子被限流:**

| 行为 | 影响程度 | 说明 |
|-----|---------|-----|
| 外链放正文 | 🔴 严重 (-25~40%) | 算法讨厌用户离开平台 |
| 发布后频繁编辑 | 🟡 中等 | 发布后 10 分钟内不要编辑 |
| 使用自动化评论工具 | 🔴 严重 | LinkedIn 已明确打击 |
| 参与互赞群 (Engagement Pod) | 🔴 严重 | 被检测到会限流 |
| 过于商业化/推销语气 | 🟡 中等 | 软性分享优于硬广 |
| 不一致的发布频率 | 🟡 中等 | 算法偏好规律发布者 |
| 纯 AI 生成内容 | 🟡 中等 | 明显 AI 味会被用户忽略 |

#### 外链处理技巧
1. **评论区放链接**: 发帖后立刻在第一条评论放链接
2. **删除预览卡**: 粘贴链接生成预览后，删掉文字链接，保留卡片
3. **用 Article 功能**: 长内容直接在 LinkedIn 发 Article

### 发布后互动策略

**黄金窗口: 发布后 30-60 分钟**

1. **立即回复**: 每一条评论都要回复，保持对话
2. **引导讨论**: 帖子末尾加开放问题
3. **主动互动**: 发帖后去互动其他人的帖子
4. **避免自嗨**: 不要自己反复评论自己

**最佳实践**:
- 发帖后 1 小时内保持在线
- 第一个评论可以自己发（补充信息或放链接）
- 用 @ 提及相关的人（但要克制，只 @真正相关的）

### API 实现建议

```
发布流程:
1. OAuth 登录获取 access_token (存储在本地)
2. 如有图片/PDF → 先上传媒体资源获取 URN
3. 调用 Posts API 发布
4. 记录 post_id 用于后续追踪
```

**无需第三方授权服务**，直接用官方 OAuth 流程最安全。
