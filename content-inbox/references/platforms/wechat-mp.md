## 第一层：技术可行性

### API 概况

**公众号 API 有两个版本**:
1. **订阅号和未认证服务号**: 基础接口（有限）
2. **已认证服务号**: 全量接口（完整功能）

**核心 API 端点**:
- **草稿箱 API**:
  - 新增草稿: `POST https://api.weixin.qq.com/cgi-bin/draft/add?access_token=TOKEN`
  - 获取草稿列表: `GET https://api.weixin.qq.com/cgi-bin/draft/batchget?access_token=TOKEN`
  - 删除草稿: `POST .../draft/delete?access_token=TOKEN`
- **发布 API**:
  - 发布草稿: `POST https://api.weixin.qq.com/cgi-bin/freepublish/submit?access_token=TOKEN`
  - 删除发布: `POST .../freepublish/delete?access_token=TOKEN`
  - 获取发布状态: `POST .../freepublish/get?access_token=TOKEN`
- **素材管理 API**:
  - 上传图片: `POST .../media/uploadimg?access_token=TOKEN`
  - 上传图文消息图片: `POST .../media/uploadimg?access_token=TOKEN`

**文档地址**: https://developers.weixin.qq.com/doc/offiaccount/

### 认证方式

**两种账号类型对比**:

| 类型 | 个人订阅号 | 企业服务号 | 已认证服务号 |
|-----|----------|----------|----------|
| **申请主体** | 个人 | 企业/组织 | 企业+认证 |
| **API 权限** | 基础 | 基础+部分高级 | 全量 |
| **费用** | 免费 | 认证费 ¥300/年 | 认证费 ¥300/年 |
| **群发次数** | 1 次/天 | 4 次/月 | 4 次/月 |
| **草稿箱 API** | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| **客服接口** | ❌ | ✅ | ✅ |
| **模板消息** | ❌ | ✅ | ✅ |
| **支付接口** | ❌ | ✅ | ✅ |

**Ming 的选择建议**:
- **个人订阅号**: 最合适，免费，每天可发，适合个人创作者
- **企业服务号**: 不适合，每月只能发 4 次，且需要企业资质

**认证方式**:
- OAuth 2.0 (server-side only)
- **Access Token**: 2 小时过期
- **Refresh Token**: 不支持自动刷新，需定时重新获取
- IP 白名单: 必须配置服务器 IP 才能调用 API

### Token 管理

```
获取 Token 流程:
1. 在公众号后台获取 AppID 和 AppSecret
2. 调用: https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=ID&secret=SECRET
3. 返回: { "access_token": "TOKEN", "expires_in": 7200 }
4. 存储: Redis/内存，定时刷新
```

**注意 ⚠️**:
- Token 是 App 级别的，不是用户级别
- **不支持 Refresh Token**
- 每分钟最多调用 10 次 token 接口
- 建议每 1.5 小时刷新一次

|-----|-----|-----|

### 能发什么 / 不能发什么

**能发**:
- 图文消息（最多 20 张图片）
- 纯文本消息
- 视频消息（需先上传素材）
- 音频消息
- 图文卡片（带缩略图）

**不能发**:
- 外链（自动屏蔽）
- 非白名单域名图片
- 诱导分享内容
- 未经授权的音频/视频

## 第二层：平台规则与最佳实践

### 内容审核机制 (2025-2026)

#### 三级审核体系

1. **机器预审**: AI 自动检测敏感词、违规内容
2. **人工复审**: 抽查或用户举报后人工审核
3. **用户举报**: 发布后用户可举报

#### 敏感词类型 (2025 最新)

| 类型 | 例子 | 后果 |
|-----|-----|-----|
| 政治类 | 领导人名字、敏感时政词汇 | 删除/封号 |
| 色情类 | 露骨描述、擦边内容 | 删除/警告 |
| 迷信类 | 算命、占卜、风水 | 限流/删除 |
| 广告类 | "最"、"第一"、"国家级"等极限词 | 限流 |
| 侵权类 | 未经授权图片、音乐、视频 | 删除 |
| 诱导类 | "分享到朋友圈" "点赞才能看" | 限流 |

**2025 新增 AI 检测**:
- AI 生成内容需标注「AI辅助创作」
- 未标注可能被限流 30-50%
- 检测方法：语义相似度 + 行文模式分析

#### 检测工具推荐

**壹伴助手**: 2025 市占率最高的检测工具
- 基于深度学习 NLP
- 可识别谐音字、变形词、隐晦表达
- 准确率达 98%，误判率 < 2%
- 支持已发布文章回溯检测

**使用方法**:
1. 安装 Chrome 插件
2. 在公众号编辑器中点击「违规检测」
3. 高亮显示敏感词并提供替换建议

### 草稿箱 API 使用最佳实践

#### 草稿箱 API 限制

```
POST https://api.weixin.qq.com/cgi-bin/draft/add?access_token=TOKEN

请求 Body:
{
  "articles": [
    {
      "title": "标题",              // 必填，最多 64 字符
      "author": "作者",             // 选填，最多 8 字符
      "digest": "摘要",             // 选填，最多 120 字符
      "content": "HTML 内容",        // 必填，最多 20000 字符
      "content_source_url": "原文链接", // 选填
      "thumb_media_id": "封面图媒体ID", // 必填
      "need_open_comment": 1,       // 1 开启评论，0 关闭
      "only_fans_can_comment": 0    // 0 所有人可评论，1 仅关注者可评论
    }
  ]
}
```

**重要限制**:
- **草稿保存上限**: 5000 条
- **单个草稿字数**: 2 万字（但实际建议 < 5000 字）
- **封面图**: 必须 900×500 像素，小于 5MB，JPG 格式
- **图片**: 必须使用微信内部图片（外链图片无法显示）

#### 图文混排方案

因为 content 是 HTML 格式，推荐格式:
```html
<p><img data-src="https://mmbiz.qpic.cn/xxx" data-ratio="0.5625" /></p>
<p>正文段落</p>
<p><img data-src="https://mmbiz.qpic.cn/yyy" /></p>
```

**图片处理流程**:
1. 先调用 `media/uploadimg` 上传图片，获取 URL
2. 在 HTML 中引用微信内部 URL
3. 不能直接用外链图片（会被屏蔽）

#### 发布工作流程

```
方案一: API + 手动确认
├── AI 生成内容
├── API 创建草稿
├── 人工检查预览
└── 手动点击群发（微信公众号后台）

方案二: 全自动发布
├── AI 生成内容
├── API 创建草稿
├── API 提交发布 (freepublish/submit)
└── 自动群发（无需人工干预）
```

**建议**: 方案一更安全，可以审核后再发

### 发文频率限制

| 账号类型 | 每日 | 每月 | 说明 |
|---------|-----|-----|-----|
| **个人订阅号** | 1 次/天 | 30 次/月 | 每次最多 8 篇文章 |
| **企业服务号** | 0.13 次/天 | 4 次/月 | 每次最多 8 篇文章 |

**重要细节**:
- "群发" ≠ "发布": 群发会推送给订阅者，发布只是生成链接
- "每日" = 自然日 00:00-23:59
- 已发布文章可以删除，但会占用当日额度（如果重新发布）

### 排版最佳实践 (2025 最新)

#### 基础规则

| 项目 | 推荐值 | 说明 |
|-----|-------|-----|


### API 实现建议

```
发布流程 (半自动):

1. 准备内容
   ├── AI 生成文章 (Markdown)
   ├── 转换为 HTML
   ├── 上传图片获取 media_id
   └── 生成封面图

2. 创建草稿 (API)
   ├── POST /draft/add
   └── 获取草稿 media_id

3. 审核预览
   ├── 在公众号后台查看草稿
   └── 人工确认排版、内容

4. 发布 (手动 or API)
   ├── 方案 A: 后台点击群发 (推荐)
   └── 方案 B: API /freepublish/submit

优势:
- 半自动: 提高效率，保留人工审核
- 安全: 避免自动发布错误
- 可控: 可以调整推送时间
```

**开发注意事项**:
1. **IP 白名单**: 必须在公众号后台配置服务器 IP
2. **错误处理**: 微信 API 返回错误码需详细日志
3. **频率控制**: 不要频繁调用，避免触发限流
4. **Token 缓存**: 2 小时刷新一次足够

**优势最大化**:
- 充分利用"搜一搜" SEO 流量
- 用草稿箱 API 实现内容批量准备
- 手动群发确保最佳发布时间
- 长期积累 = 稳定流量来源
