## 第一层：技术可行性

### API 概况
- **官方 API**: X API v2
- **文档地址**: https://developer.x.com/
- **核心端点**:
  - 发推: `POST https://api.twitter.com/2/tweets`
  - 上传媒体: `POST https://upload.twitter.com/1.1/media/upload.json`
  - 删除: `DELETE https://api.twitter.com/2/tweets/:id`

### 认证方式
- **OAuth 2.0 PKCE** (推荐) 或 OAuth 1.0a
- 需要在 X Developer Portal 创建项目和应用
- 所需 Scope:
  - `tweet.read` - 读取推文
  - `tweet.write` - 发布推文
  - `users.read` - 读取用户信息
  - `offline.access` - 获取 refresh token

### Token 管理
- **Access Token 有效期**: 2 小时
- **Refresh Token**: 支持，需要 `offline.access` scope
- 必须实现自动刷新机制
- Free tier 不支持某些 token 操作

### 免费额度 (Free Tier)

**⚠️ X API Free Tier 限制非常严格 (2025-2026):**

| 限制项 | 数值 | 说明 |
|-------|-----|------|
| 发帖上限 | **500 帖/月** | App 级别限制 |
| 每日发帖 | ~17 帖/天 | 500÷30 |
| 读取 API | **几乎不可用** | 1 次请求/15分钟 |
| 用户查找 | 1 次/24小时 | 极度受限 |
| 搜索功能 | ❌ 不可用 | 需要 Basic ($100/月) |

**结论**: Free tier **只能写，不能读**，足够个人日常发帖（每天 2-3 条），但无法做任何数据分析。

### 付费 Tier 对比

| Tier | 价格 | 发帖 | 读取 | 适用场景 |
|------|-----|-----|-----|---------|
| Free | $0 | 500/月 | 几乎无 | 个人发帖 |
| Basic | $100/月 | 3,000/月 | 10K/月 | 小型应用 |
| Pro | $5,000/月 | 无限 | 1M/月 | 商业应用 |
| Enterprise | $42K/月 | 无限 | 50M/月 | 大型企业 |

### 能发什么 / 不能发什么

**能发**:
- 文本推文 (最长 280 字符，Premium 用户 25,000 字符)
- 图片 (最多 4 张)
- 视频 (最长 140 秒 / 512MB)
- GIF
- 投票 (Poll)
- Thread (串联推文)
- 引用转推

**Free Tier 不能**:
- 读取他人推文
- 搜索推文
- 获取 timeline
- 批量操作

## 第二层：平台规则与最佳实践

### 算法偏好 (2025-2026)

X 算法经历重大变化，核心是 **"付费优先 + Grok AI 驱动"**

#### 2026 算法核心逻辑

1. **For You Feed 主导**: 默认显示算法推荐，Following 只是二级选项
2. **X Premium 特权**: 付费用户获得 **2-4 倍 reach 加成**
3. **SimClusters 分类**: Grok AI 根据内容语义将用户分入"兴趣社区"
4. **时间衰减**: 帖子每 6 小时损失一半曝光潜力

#### 互动信号权重 (基于开源代码分析)

| 行为 | 权重倍数 | 说明 |
|-----|---------|-----|
| 转发 (Repost) | **20x** | 最重要的信号 |
| 回复 (Reply) | **13.5x** | 质量评论很有价值 |
| 收藏 (Bookmark) | **10x** | "沉默的喜欢" |
| 点赞 (Like) | **1x** | 基准线 |

**关键洞察**: 一个高 bookmark + 高 repost 的帖子，远比一个高 like 帖子表现好。

#### 外链惩罚

⚠️ **X 对外链非常敌视**:
- 包含外链的帖子 reach 下降 **50-90%**
- Elon Musk 公开确认这一机制
- 平台目标是留住用户

### 最佳发布时间

**通用最佳时间 (2026 数据)**:
- **最佳**: 周二/周三/周四，上午 9-11 AM (受众当地时间)
- **次选**: 工作日 10AM - 5PM
- **最差**: 周末

**Tech/AI 受众特点**: 工作时间最活跃

**Ming 在悉尼的策略**:

| 目标 | 悉尼发帖时间 | 覆盖区域 |
|-----|------------|---------|
| 美西 Tech | 凌晨 2-4 AM | 硅谷早晨 |
| 美东 | 晚上 10PM - 凌晨 1AM | 纽约早晨 |
| 欧洲 | 下午 5-8PM | 欧洲早晨 |
| 亚太 | 早上 9-11AM | 本地时间 |

**建议**: 轮流覆盖不同时区，或选悉尼晚上 10 点 (美东早晨)

### 内容格式对比

| 格式 | 互动率 | reach | 建议 |
|-----|-------|------|-----|
| **Thread (串推)** | 2-4% | 最高 (+60% profile visits) | **首选**，深度内容最佳载体 |
| 带图推文 | 2-3x 基准 | 高 | 视觉吸引力 |
| 视频 | 中等 | 中 | X 在推视频，原生上传 |
| 纯文本 | 0.5-1.5% | 中低 | 快速观点/段子 |
| 带外链 | 低 | **-50~90%** | 尽量避免 |

#### Thread 最佳实践
- **长度**: 3-7 条推文最佳
- **首条**: Hook，必须吸引人
- **每条**: 独立完整，可被单独转发
- **末条**: 总结 + CTA + Hashtag

### Hashtag 策略

**2025-2026 Hashtag 现状**:
- X 仍是 hashtag 最有效的平台
- 但权重低于内容质量

**使用建议**:
- **数量**: 2-3 个最佳
- **位置**: Thread 最后一条，或正文自然嵌入
- **类型**: 
  - 1 个宽泛 (#AI, #Tech)
  - 1-2 个精准 (#AIAgents, #LLMs)
- **避免**: 堆砌 hashtag，看起来像垃圾营销

**Tech/AI 有效 hashtag**:
```
#AI #MachineLearning #LLMs #OpenAI #Claude #BuildInPublic #IndieHacker #TechTwitter
```

### 降权行为 (Shadowban 触发)

| 行为 | 风险等级 | 后果 |
|-----|---------|-----|
| 外链 | 🔴 高 | Reach 下降 50-90% |
| 自动化发帖 | 🔴 高 | 账号标记为 bot |
| 大量快速关注/取关 | 🔴 高 | Shadowban |
| 重复内容 | 🟡 中 | 被判定为 spam |
| 使用敏感词 | 🟡 中 | 内容被隐藏 |
| 频繁被 block | 🟡 中 | 可信度下降 |
| 未验证账号 | 🟡 中 | 回复不易被看到 |

#### Shadowban 类型
1. **Search Ban**: 推文不出现在搜索结果
2. **Search Suggestion Ban**: 账号不出现在搜索建议
3. **Reply Barrier**: 回复被折叠

**检测工具**: https://shadowban.yuzurisa.com/

### 互动策略

#### 回复是增长利器
> "一个在热门推文下的优质回复，可能比一周的原创帖带来更多曝光"

**策略**:
1. **抢早回复**: 在大 V 发帖后 10 分钟内回复
2. **添加价值**: 不要只说 "Great post"，补充观点
3. **带幽默**: Tech Twitter 爱 meme 和 wit

#### Quote Tweet vs Repost
- **Quote Tweet**: 添加你的观点，更有价值
- **Repost**: 纯转发，权重稍低

#### 自我 Thread
发完首条后，用 Reply 继续发后续内容，形成 Thread

## 对我们的建议

### 针对 Ming 的定位

**X 对 Ming 的价值**:
- 全球 AI/Tech 社区聚集地
- 最快的行业信息流
- 英文为主，国际曝光

**建议定位**: AI 工程师视角 + 创业实战 + 悉尼国际视角

### 内容策略建议

1. **主打 Thread**: 技术内容做成 5-7 条的 Thread
   - 例: "I built an AI agent in a weekend. Here's exactly how (thread 🧵)"

2. **每周节奏**:
   - 每周 2-3 个 Thread (深度)
   - 每天 1-2 条快速观点/分享 (轻量)
   - 每天花 15 分钟回复大 V 帖子

3. **时间策略**: 
   - 悉尼晚上 10 点发帖 (覆盖美东早晨)
   - 偶尔早上 9 点 (覆盖亚太)

4. **外链处理**:
   - 正文绝不放链接
   - 链接放 Thread 最后一条的回复里
   - 或者用 X 的 Article 功能 (长文)

5. **语言**: 全英文

### API 实现建议

**Free Tier 足够 Ming 的用量**:
- 每天 2-3 条推文 × 30 天 = 60-90 条 < 500 限制 ✅

```
发布流程:
1. OAuth 2.0 PKCE 获取 token
2. 如有图片 → media/upload 先上传
3. POST /2/tweets 发布
4. Thread: 用 reply_to_tweet_id 串联
5. 存储 tweet_id 用于追踪
```

**注意**: Free tier 无法读取 analytics，需要手动在 X 网页端看数据

### X Premium 是否值得？

| 考虑因素 | 分析 |
|---------|-----|
| 价格 | ~$8-11/月 (地区不同) |
| Reach 提升 | 2-4 倍 |
| 编辑推文 | ✅ 有用 |
| 长推文 | 25K 字符 vs 280 |
| 回复优先展示 | ✅ 很有价值 |

**建议**: 如果认真做 X，Premium 值得考虑（Reach 提升明显）
