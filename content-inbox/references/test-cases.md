# Content Inbox 测试用例

## 测试目标

验证 content-inbox skill 是否正确工作：
1. 检测抖音链接
2. 调用 douyin-downloader 下载
3. 询问用户处理方式
4. 根据配置自动执行或询问

---

## 测试用例

### 测试 1：首次发链接（无配置）

**输入**：
```
https://v.douyin.com/vH931iPlYbU/
```

**期望输出**：
```
✅ 已下载：[标题]
- 作者：xxx
- 时长：12 分 48 秒

要怎么处理？
A. 仅下载（已完成）
B. 转写文字（~5 分钟）
C. 完整分析（转写+验证+素材化，~20 分钟）
D. 生成博客草稿（~30 分钟）
```

**验证点**：
- [ ] 检测到抖音链接
- [ ] 调用 douyin-downloader
- [ ] 下载成功
- [ ] 询问用户

---

### 测试 2：用户选择 B（转写）

**输入**：
```
B
```

**期望输出**：
```
✅ 开始转写...
⏳ 预计需要 5 分钟，完成后通知你
```

**验证点**：
- [ ] 后台启动 SubAgent
- [ ] 主会话不阻塞
- [ ] 5 分钟后通知完成

---

### 测试 3：第三次发链接（3 次相同操作）

**前置条件**：
- 用户已连续 2 次选择 C

**输入**：
```
https://v.douyin.com/yyy/
```

**期望输出**：
```
✅ 已下载：[标题]

[预测] 完整分析？
你已连续 3 次选择 '完整分析'。是否设为默认？
- 全局
- 仅本项目
- 不设置
```

**验证点**：
- [ ] 检测到 3 次相同操作
- [ ] 询问是否设为默认
- [ ] 提供选项

---

### 测试 4：用户说"Always use C"

**输入**：
```
https://v.douyin.com/zzz/
Always use full analysis for douyin
```

**期望输出**：
```
✅ 已下载：[标题]
✅ 已确认偏好：douyin_default_action = full_analysis（明确表达）

开始完整分析...
```

**验证点**：
- [ ] 检测到明确偏好
- [ ] 直接写入 GLOBAL
- [ ] 自动执行 C

---

### 测试 5：项目特定配置

**前置条件**：
- 当前项目：accounting-ai
- projects/accounting-ai/config.md 有配置：douyin = D

**输入**：
```
https://v.douyin.com/aaa/
```

**期望输出**：
```
✅ 已下载：[标题]
[项目配置] accounting-ai：生成博客草稿

开始生成博客草稿...
```

**验证点**：
- [ ] 检测到项目配置
- [ ] 应用 PROJECT 层配置
- [ ] 自动执行 D

---

### 测试 6：临时偏好

**前置条件**：
- GLOBAL 配置：douyin = C

**输入**：
```
https://v.douyin.com/bbb/
这次只下载
```

**期望输出**：
```
✅ 已下载：[标题]
[临时偏好] 本次只下载，不分析

完成！
```

**验证点**：
- [ ] 检测到临时偏好
- [ ] 写入 SESSION 层
- [ ] 覆盖 GLOBAL 配置

---

### 测试 7：第五次相同操作（快速通道）

**前置条件**：
- 用户已连续 4 次选择 C（7 天内）

**输入**：
```
https://v.douyin.com/ccc/
C
```

**期望输出**：
```
✅ 已下载：[标题]
✅ 已自动确认偏好：douyin_default_action = full_analysis（5 次相同操作）

开始完整分析...
```

**验证点**：
- [ ] 检测到 5 次相同操作
- [ ] 自动写入（无需确认）
