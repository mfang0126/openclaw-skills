# X / Twitter 单条推文适配模板

## 格式要求
- **字数限制：** 280 字符（硬限制）
- **甜区：** 70-140 字符（简短有力）
- **语言：** 英文
- **Free tier 限制：** 500 帖/月

## 结构

```
[一句犀利观点 or hot take]

[可选：补充一句]

[可选：1-2 个 hashtag]
```

## 平台特色元素
- ✅ 有争议的观点（引发 quote tweet）
- ✅ 数字和具体细节（"saved 6 hours/week"）
- ✅ 问问题（引发回复）
- ✅ 截图/代码截图（提高停留时间）
- ❌ 不堆 hashtag（0-2 个就够，多了像 spam）
- ❌ 不放纯链接推文（算法降权）
- ❌ 链接放回复里

## 发布时间（悉尼时区）
- 最佳：AEST 早 6-8 点（= 北美前一天下午 3-5 点 ET）
- 次佳：AEST 晚 10-11 点（= 北美早上 7-8 点 ET）
- 最佳日：周三

## 发布后 30 分钟
**必须在线互动** — 算法重度看早期参与：
- 回复所有评论
- 自己回复补充上下文
- 如果有人 quote tweet，去互动

## 示例

```
Hot take: AI agents don't need complex memory systems.

A simple markdown file that gets injected on session start beats any vector DB for personal agents.

I've been running this for 3 months. Zero retrieval failures.
```

```
Spent $0 on AI agent hosting.

Mac Studio + OpenClaw + cron jobs.

Runs 24/7. No cloud. No subscriptions. All data stays on my machine.
```
