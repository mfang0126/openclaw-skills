# v2.1 更新：针对 Telegram 优化

**更新日期**：2026-02-07  
**更新原因**：用户观察到"Telegram 需要的更多"

---

## 🎯 核心改进

### Telegram 特殊处理（更主动）

**之前（v2.0）**：
- 所有渠道统一规则
- 只对"重要视觉修改"截图

**现在（v2.1）**：
- **Telegram 更主动**：任何视觉变化都截图
- **浏览器/Terminal 适度**：只对重要视觉修改截图

---

## 📊 对比示例

| 修改 | Telegram | 浏览器/Terminal |
|------|----------|----------------|
| `color: red → blue` | ✅ 截图 | ✅ 截图 |
| `margin: 10px → 12px` | ✅ 截图 ⭐ | ❌ 不截 |
| `标题：Hello → Hi` | ✅ 截图 ⭐ | ❌ 不截 |
| `border: 1px → 2px` | ✅ 截图 ⭐ | ❌ 不截 |
| `<button>新增</button>` | ✅ 截图 | ✅ 截图 |
| `<!-- 注释 -->` | ❌ 不截 | ❌ 不截 |

⭐ = v2.1 新增截图的场景

---

## 💡 为什么这样设计？

### Telegram 的特殊性

1. **用户看不到本地文件**
   - 浏览器用户：可以刷新预览
   - Terminal 用户：可以 `open` 文件
   - Telegram 用户：只能靠截图 ⭐

2. **完全依赖截图**
   - 浏览器/Terminal：截图是辅助
   - Telegram：截图是唯一方式

3. **宁可多不可少**
   - 微调（margin 10→12px）也截图
   - 小文本改动也截图
   - 避免用户看不到效果

---

## 🔧 技术实现

### 判断逻辑

```javascript
const channel = runtime.channel

if (channel === "telegram") {
  // Telegram: 更宽松
  shouldScreenshot = (
    任何样式变化 ||
    任何布局变化 ||
    任何文本改动 ||
    加/删元素
  ) && !(注释 || 格式化)
  
} else {
  // 浏览器/Terminal: 更严格
  shouldScreenshot = (
    重要颜色变化 ||
    重要布局变化 ||
    加/删元素
  )
}
```

> **不同渠道，不同需求**  
> Telegram 用户完全依赖截图 → 更主动  
> 浏览器/Terminal 用户可自行预览 → 更克制
