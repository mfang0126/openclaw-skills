# 安全策略

## 频率限制

| 规则 | 限制 |
|------|------|
| 每日最大发帖数 | 5 篇 |
| 两篇最小间隔 | 2 小时 |
| 禁止发帖时段 | 00:00 - 06:00 |
| 发布时间随机偏移 | ±30 分钟 |

频率限制为硬性约束，不可跳过。检查时读取 `state.json` 中今日记录数量和上次发帖时间。

偏移由 publish.py 在执行前自动计算并应用，AI 和用户不需要手动处理。

## 超发模式（Overshoot）

当今日已发布 5 篇，用户仍要求继续发布时，AI **必须**发出以下警告：

> ⚠️ 今日已自动发布 5 篇，继续发布可能触发平台风控。额外发布带来的限流/封号风险由你自行承担。确定要继续发布吗？

规则：
- AI **绝不**自动发布超过 5 篇的帖子
- 用户必须**明确确认**后才能继续（"发"、"确认"、"继续"等）
- 用户取消或未明确确认 → 停止，不执行
- 每次超发请求都需要重新警告和确认

## 人审门控

发布前的强制确认流程：

1. AI 生成草稿（标题 + 正文 + 标签）
2. 发给用户预览
3. 用户明确说"发"/"确认"/"ok"才执行
4. 用户说改 → 修改后再确认
5. 用户说取消 → 停止，不执行任何操作

**核心规则：不经过用户确认，绝不调用任何发布命令。**

## 行为模拟

publish.py 包装脚本自动执行以下人类行为模拟：

1. **预热**：浏览 3-5 篇笔记 + 点赞 1-2 篇（约 2-3 分钟）
2. **发帖前延迟**：15-30 秒（模拟打开 App 浏览）
3. **犹豫延迟**：5-15 秒（模拟"要不要发"的思考）
4. **调用 sau 执行发布**
5. **发帖后延迟**：10-25 秒（检查是否发布成功）
6. **顺手刷推荐页**：15-40 秒

## 状态追踪

状态文件：`~/.openclaw/skills/xhs-publisher/state.json`

```json
{
  "posts": [
    {
      "date": "2026-04-01",
      "time": "10:30",
      "type": "note",
      "title": "...",
      "tags": ["tag1", "tag2"],
      "images": 1
    }
  ]
}
```

每次发帖后必须更新此文件。字段说明：
- `date`：发布日期（YYYY-MM-DD）
- `time`：发布时间（HH:MM）
- `type`：笔记类型（`note` 或 `video`）
- `title`：发布标题
- `tags`：使用的标签列表
- `images`：图片数量（图文笔记时）
